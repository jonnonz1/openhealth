# specs

Durable design documents for `openhealth`. Each spec is numbered, dated, and written to be readable by a future contributor (or LLM) with zero prior context.

## Conventions

- `NNN-short-title.md` — three-digit sequence, kebab-case title.
- Start with a **Context** section (why this exists).
- End with a **Verification** section (how we know it's done).
- Supersede rather than edit: if a spec materially changes, write a new spec and mark the old one `Status: superseded by NNN`.
